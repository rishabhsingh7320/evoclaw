How it works (behind the scenes)
  1. You send a query → API creates a session (if new) and a run, saves your message, and starts the orchestrator in the background.
  2. Orchestrator loop → For each step, the Manager agent decides: answer directly (RESPOND) or call a tool (e.g. search, math, file).
  3. Tool calls → If a tool is chosen, it runs (e.g. fake search, calculation); the result is stored and then turned into an assistant reply.
  4. Events → Every action (run_started, step_started, tool_started, tool_finished, assistant_message, run_finished) is written to the DB and pushed over WebSocket to this page.
  5. Conversation → When an assistant_message event is received, its content is shown in the left panel.
